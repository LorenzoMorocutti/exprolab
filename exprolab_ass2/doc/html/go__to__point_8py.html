Publishes to:

Subscribes to:
/sub_odom(nav_msgs.msg.Odometry) /sub_vel(geometry_msgs.msg.Twist)

ServiceServer:
none

ActionServer:
/go_to_point (exprolab_ass2.action.Moving)

Description:

The node controls the behavior of the robot according to the go_to_point behavior via an action server. A FSM is used to model the behavior every time a new goal pose is received: firstly the robot alight its position with the goal's one, then it goes straight to the goal position, it aligns with the goal orientation and then the goal pose is reached.
